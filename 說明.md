初始專案
1.把檔案放進建立好的github儲存庫
2.打標題說明
    【github 前綴說明】
        init:初始化專案
        feat:新增功能或畫面等
        fix:修改
        style:樣式
        doc:說明文件和相關
3.Summary標題要寫, 說明內容可寫可不寫
4.Commit -> 上傳到暫存空間
5.publish branch -> 同步到github
6.fetch orgin -> 同步成功

更新專案
1.先確認儲存庫有沒有檔案要先同步(點一下fetch origin)
    -如需更新, 需把新的檔案拉下來 -> pull origin
        -如跳出error => 有衝突(新檔案和你的檔案有改寫到同樣的地方)
        -把自己寫的東西放到暫存空間 -> stash changes and continue
        -多點幾次fetch確保沒有人在上傳
        -把自己的內容放回去 -> stashed changes(在summary的上面) -> restore(點discard會直接消失)
        -檔案顯示紅色三角形表示有衝突需解決 -> 右鍵在VSC裡打開看要保留哪一個版本
    *!只要有綠色勾勾就一定要上傳!*
2.打Summary標題(Description描述看心情)
3.Commit -> 上傳到暫存空間
4.push origin -> 同步到github
5.fetch orgin -> 同步成功
